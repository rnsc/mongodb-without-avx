# MongoDB without AVX - Build System Notes

## Project Goal
Build MongoDB 8.0.x from source without AVX/AVX2/AVX512 CPU instructions, so it
runs on older hardware (e.g., Intel Atom, pre-2011 CPUs, virtualized environments
without AVX passthrough). The target runtime is Debian 12 (Bookworm).

**Supported versions:** 8.0.x only (8.0.13+ uses Bazel; earlier uses SCons).
The Dockerfile validates the version and rejects non-8.0.x values.

## Repository Structure
- `Dockerfile` — main build (Bazel, MongoDB 8.0.x)
- `CLAUDE.md` — this file (build system notes for AI assistants)

## Build Command
```bash
# Simple build (auto-detect CPU count)
docker build --progress=plain -t mongo-wo-avx:8.0.19 .

# With explicit job count (recommended: leave 1 core free)
docker build --build-arg NUM_JOBS=11 --progress=plain -t mongo-wo-avx:8.0.19 .

# Different MongoDB version
docker build --build-arg MONGO_VERSION=8.0.18 --progress=plain -t mongo-wo-avx:8.0.18 .
```

Build takes ~90 minutes with 11 jobs on a Ryzen 5 9600X (24GB RAM).

## AVX Patches Applied

All patches achieve **zero AVX and zero VEX-encoded instructions** in the final binary.

### 1. Compiler Flags (`mongo_compiler_flags.bzl`)
Replaces `-march=sandybridge` (implies AVX) with `-march=x86-64-v2` (SSE4.2, no AVX).
Also adds global `--copt=-mno-avx -mno-avx2 -mno-avx512f` and `--per_file_copt`.
Note: `--copt` can be overridden by target-level `copts`, which is why source patches
are also needed.

### 2. mozjs (`BUILD.bazel` + `SIMD_avx2.cpp`)
Removes `-mavx2` from `PLATFORM_COPTS` and `select()` blocks.
Replaces `SIMD_avx2.cpp` with stub implementations returning `nullptr`.
The AVX2 SIMD functions are only called after a runtime CPU check, so stubs are safe.

### 3. CRoaring (`BUILD.bazel`)
Adds `-DROARING_DISABLE_X64` to disable AVX2/AVX512 intrinsics.
CRoaring uses `__attribute__((target("avx2")))` which bypasses `-mno-avx2`,
so the define is the only way to disable it.

### 4. Toolchain libstdc++ (fetch + replace)
MongoDB's hermetic toolchain v4 bundles a `libstdc++.a` compiled with VEX-encoded
instructions (AVX using xmm registers). The build fetches the toolchain with
`--nobuild`, then replaces its `libstdc++.a` and `libstdc++.so` (a linker script
referencing the .a) with a symlink to the system's `/usr/lib/x86_64-linux-gnu/libstdc++.so.6`.
Debian 12's shared libstdc++ has zero VEX instructions.

## Build System Fixes

### 5. Enterprise Module Stubs
~46 stub `BUILD.bazel` files with empty `filegroup` targets, generated by a Python
script. Required because the community source references enterprise packages.
If upgrading version, re-check `src/BUILD.bazel` for new enterprise paths.

### 6. Toolchain Include Path (`mongo_toolchain_flags_v4.bzl`)
Adds `/src` to `COMMON_BUILTIN_INCLUDE_DIRECTORIES` to fix absolute path
inclusion errors when building in Docker with `WORKDIR /src`.

### 7. Build Environment
- `--action_env=LD_PRELOAD=...libstdc++.so.6` — forces system libstdc++ during build
- `--action_env=SSL_CERT_*` — fixes TLS errors in Bazel's sandboxed Python actions
- `--define=MONGO_VERSION/GIT_COMMIT_HASH` — required for tarball builds (no .git)
- `--fission=no` — disables split DWARF (fixes mozjs .dwo errors)
- `--config=opt --config=local` — opt must come before local to override remote exec

### 8. Binary Discovery
Build and binary copy are separate `RUN` steps for cache-friendly iteration.
Use `find -L` and `cp -L` to follow Bazel's symlinks into `~/.cache/bazel`.

## Toolchain
- MongoDB's hermetic `mongo_toolchain_v4` (clang-based, GCC 11.3.0 sysroot)
- Specified via `--//bazel/config:mongo_toolchain_version=v4`
- v5 is NOT available for 8.0.x — only v4 is allowed
- Downloaded automatically by Bazel during the build

## Checking for AVX in Built Binary
```bash
# Check the embedded report
docker run --rm --entrypoint cat mongo-wo-avx:8.0.19 /avx_report.txt

# Full scan (requires binutils)
docker run --rm --entrypoint bash --user root mongo-wo-avx:8.0.19 -c \
    "apt-get update -qq && apt-get install -yqq binutils > /dev/null 2>&1 && \
     echo 'ymm:' && objdump -d /usr/local/bin/mongod | grep -cE '%ymm' && \
     echo 'vex-xmm:' && objdump -d /usr/local/bin/mongod | grep -cP '\tv[a-z].*%xmm'"
```

## Files to Check When Upgrading MongoDB 8.0.x Patch Version
1. `src/BUILD.bazel` — enterprise package references (update stub list if changed)
2. `bazel/toolchains/cc/mongo_linux/mongo_compiler_flags.bzl` — march/AVX flags
3. `src/third_party/mozjs/BUILD.bazel` — mozjs AVX flags
4. `src/third_party/croaring/BUILD.bazel` — CRoaring AVX intrinsics
5. New third-party libraries with AVX — a dedicated Dockerfile can be created to
   scan source BUILD files for AVX flags without running a full build

## Runtime Image
- Base: `debian:12-slim`
- Includes: `mongod`, `mongos`, `mongosh` (prebuilt), `/avx_report.txt`
- Runtime deps: `libcurl4`, `libssl3`, `liblzma5`, `libstdc++6`, `ca-certificates`
- MongoDB user: uid/gid 999
- Data: `/data/db`, `/data/configdb` (volumes)
- Port: 27017
